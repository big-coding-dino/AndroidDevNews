## March 2026

### [Android Bench: Google's Public LLM Leaderboard for Android Dev](https://android-developers.googleblog.com/2026/03/elevating-ai-assisted-androi.html)
Google launched Android Bench, the first public benchmark that scores LLMs specifically on Android tasks: resolving breaking API changes across releases, domain-specific challenges like Wear OS networking, and Jetpack Compose migrations. Models are measured by whether they can actually fix reported issues from real public GitHub Android repos, verified by unit or instrumentation tests — not synthetic or self-reported evals. The current spread is stark: models ranged from 16% to 72% completion, with Gemini 3.1 Pro topping the leaderboard and Claude Opus 4.6 close behind. The methodology and dataset are public on GitHub, and Google embedded canary strings to discourage training-data contamination. Practically, it gives you a data-driven basis for choosing your AI assistant for Android work — not just general coding benchmarks where Android specifics are invisible.

### [Android Studio Panda 2: Prompt → Deployed App](https://android-developers.googleblog.com/2026/05/go-from-prompt-to-working-prototype.html)
The new "Create with AI" flow takes a text description (plus optional design screenshots), produces a full project plan for you to approve, then enters an autonomous loop: generates the code, builds it, reads compile errors, self-corrects, and repeats until it succeeds — then deploys to the emulator and walks through each screen to verify against your original prompt. Everything generates in Kotlin + Compose + latest stable libraries by default. The free-tier model works standalone, but connecting a paid Gemini API key unlocks Gemini 3.1 Pro (significantly better for agentic workflows) and a "Nano Banana" image generation model for UI design ideation. The release also includes a Version Upgrade Assistant: right-click your version catalog, select AI → "Update Dependencies," and it iteratively resolves conflicts across library upgrades without manual intervention.

### [6 Expert Tips for Gemini in Android Studio](http://android-developers.googleblog.com/2026/03/supercharge-your-android-development.html)
Six Google engineers and GDEs share concrete workflow advice rather than feature-tour content. The most actionable: (1) Trevor Johns — use an AI Studio API key to get Gemini 3.1 Pro; the default free model underperforms for agentic tasks. (2) Alejandra Stamato (HubSpot GDE) — give the agent explicit constraints and tell it not to invent solutions for experimental APIs. (3) Jose Alcérreca — use "search the official documentation" prompts in Agent Mode and add MCP servers for non-Android libraries to kill hallucinations. (4) Joe Birch (Buffer GDE) — create `Agent.md` files at project root or in specific modules to document your custom frameworks and design systems; the agent reads them before acting. (5) Diego Perez — offload commit messages, test generation (it mirrors your existing test style), and parser writing. (6) Trevor Johns again — use `.aiexclude` glob files to keep sensitive files out of model context on the free tier (paid plans don't train on your code at all).

### [Gemini CLI + Code Maat: Automated Tech Debt Detection](https://www.maiatoday.net/p/android-%EF%B8%8F-gemini-cli-tech-debt-detection-code-maat-skill/)
Maia Grotepass builds a Gemini CLI skill that automates Code Maat — a Java forensic tool by Adam Tornhill that analyzes git history (not just current code) to find hotspots, churn, and logical coupling between files. The skill orchestrates everything: generates the full git log, runs Code Maat analyses across four categories (summary, revisions, authors, coupling), processes the CSV output via a Python orchestrator, produces a `REPORT.md` with the top 10 findings per category, and renders an interactive D3.js visualization. On the Habitica Android repo, it surfaced 779 revisions to `strings.xml` across 42 authors and 100% logical coupling between three `RecyclerViewAdapter` classes — the kind of structural risk that no static analysis tool finds. Install once with `gemini extensions install`, ask "tell me about the hotspots in this codebase," and it just runs.

## February 2026

### [AppFunctions: Your App's Interface to AI Agents](https://android-developers.googleblog.com/2026/02/the-intelligent-os-making-ai-agents.html)
The biggest architectural shift of Q1. AppFunctions lets apps declare self-describing capabilities via the Jetpack AppFunctions library — Android's version of MCP, but executed locally on-device rather than through a cloud server. AI agents (starting with Gemini) discover these functions, invoke them, and return results without the user ever opening the app. The Samsung Galaxy S26 demo is concrete: say "show me photos of my cat from Samsung Gallery," and Gemini identifies the Gallery's AppFunction, retrieves the results, and lets you share them via text — all within Gemini's interface. Google currently has Calendar, Notes, and Tasks wired up with partners. The paradigm shift for developers: success is no longer "user launches your app," it's "agent completes the task on behalf of the user." Broader AppFunctions documentation is promised later in 2026; start watching Jetpack release notes.

### [Koog x ACP: Connect a Kotlin Agent to IntelliJ AI Chat](https://blog.jetbrains.com/ai/2026/02/koog-x-acp-connect-an-agent-to-your-ide-and-more/)
The Agent Client Protocol (ACP) is an open standard with SDKs in Python, TypeScript, Kotlin, and Rust — any ACP-compatible agent can talk to any ACP-compatible editor. Koog now ships built-in ACP support via its feature system, requiring two interfaces: `AgentSupport` (initialize, createSession, loadSession) and `AgentSession` (prompt, cancel). The agent is packaged as a distribution binary via Gradle's `installDist`, then registered in `~/.jetbrains/acp.json` — one config file, no custom plugin required. From that point, IntelliJ's AI Chat talks to your Kotlin agent with full file read/edit permission flow surfaced in the IDE UI. Requires IntelliJ 25.3+. The full working example is in the Koog repo under `examples/acp-agent`.

## January 2026

### [Android Studio Otter 3: Bring Your Own LLM](https://android-developers.googleblog.com/2026/01/llm-flexibility-agent-mode-improvements.html)
The BYOM (Bring Your Own Model) feature finally unlocks model choice: connect any OpenAI-compatible remote endpoint (GPT, Claude, anything else) via API key, or run a local model through LM Studio or Ollama for fully offline, private development. Gemini stays the default and remains the most Android-optimized option. Beyond model choice, Agent Mode got a major upgrade: it can now deploy to a connected physical device, read the screen, take screenshots, and check Logcat — closing the build-run-debug loop autonomously. New in this release: multiple conversation threads (separate contexts, persistent across sessions), MCP server connectivity (Figma, Notion, Linear, Canva are the named integrations), natural-language Journey tests (the agent records actions with reasoning screenshots and produces resilient UI tests), and an App Links assistant. The most feature-dense Studio release in memory.

### [Building AI Agents in Kotlin — Parts 4 & 5](https://blog.jetbrains.com/ai/2026/01/building-ai-agents-in-kotlin-part-4-delegation-and-sub-agents/)
**Part 4 — Delegation:** The core pattern is agent-as-a-tool: a Find sub-agent (running GPT-4 Mini for cost efficiency) handles all directory exploration and search, while the main agent retains write and execution capabilities. Koog wraps this with `createAgentTool()` in three lines. The real win is context isolation — dead-end search paths stay in the sub-agent's history, never polluting the main agent's context window. Benchmarked results: cost dropped from $1.63 to $1.47 per run (10% reduction), success rate ticked from 56% to 58%. Sub-agent model choice matters enormously — accidentally using GPT-5 Codex there spiked cost to $3.30/run.

**Part 5 — Forgetting:** Compression triggers at 200 messages *or* 200K characters (whichever hits first), configurable per workload. Two strategies: `WholeHistory` (LLM summarizes everything into one message — fast but lossy) and `RetrieveFactsFromHistory` (you define `Concept` objects — targeted questions like "What is the structure of this project?" — and the LLM answers each independently against the full history before compressing). One question per LLM call consistently outperforms bundling multiple questions. The retrieval model can be a cheaper/faster model than your main agent. JetBrains uses eight concepts for their test coding agent.

### [The AI Coding Ladder](https://fragmentedpodcast.com/episodes/301/)
Kaushik Gopal and Iury Souza map the current AI coding landscape into four distinct paradigms that correspond to genuinely different mental models: **Gen 1** — super autocomplete (Copilot, Cursor Tab); **Gen 2** — chat-oriented programming (Cursor IDE, Firebender); **Gen 3** — agents that reason and use tools autonomously (Claude Code, Junie, Jules), with three sub-flavors: TUI, IDE-integrated, and headless; **Gen 4** — agent orchestration via git worktrees for parallel independent workstreams. Practical advice: Kaushik prompts agents to "confirm if my requirements are clear" before they start executing. Iury uses compact context-transfer protocols to hand off state between agents without losing continuity.

### [Gemini 3 Flash on Firebase AI Logic](https://android-developers.googleblog.com/2025/12/build-smarter-apps-with-gemini-3-flash.html)
Gemini 3 Flash is the production-grade sibling to 3 Pro — strong at reasoning, multimodal, and tool use but optimized for low latency and cost. Firebase integration is three Kotlin lines: `Firebase.ai(backend = GenerativeBackend.googleAI()).generativeModel("gemini-3-flash-preview")`. Beyond the model itself, Firebase added two production tools worth noting: an AI Monitoring Dashboard (latency, success rates, and costs broken down per model) and Server Prompt Templates — prompts stored server-side rather than in the APK, which prevents extraction and means you can iterate on prompts without a new app release. Good fit for any app where 3 Pro's cost would be a problem in production.

### [Android AI Sample Catalog](https://android-developers.googleblog.com/2025/12/explore-ai-on-android-with-our-sample.html)
Google shipped a dedicated open-source app covering the full breadth of Android AI APIs: on-device text summarization with Gemini Nano (ML Kit GenAI API), image generation with Imagen, conversational image editing with the Nano Banana / Gemini 3 Pro Image model, image descriptions via Gemini Nano, and voice-driven to-do list management using the Gemini Live API. ## November 2025

### [Does AI Generate Accessible Android Apps?](https://eevis.codes/blog/2025-11-25/does-ai-generate-accessible-android-apps/)
Eevis Panula ran the same app prompt through Gemini, Junie, Cursor, and Claude, then tested each output with assistive technologies. Every single tool produced redundant content descriptions — a button labeled "Add yarn" getting a description of "Add new yarn," forcing screen reader users to hear the same information twice. Tool-specific failures: Gemini added a redundant focusable modifier causing focus navigation to skip elements; Junie assigned invisible components to the keyboard tab order and gave incorrect semantic roles to components; Cursor generated non-scrollable screens with poor large font support; Claude hallucinated semantic properties (`Role.TextField` doesn't exist) and appended redundant action descriptions that screen readers read twice. The broader finding is that AI tools are reproducing the same accessibility mistakes seen in human-written code — they haven't learned to do better. If accessibility matters in your app, treat AI-generated UI code as a first draft that requires an accessibility audit, not a finished implementation.

### [Koog 0.5.0: Non-Graph Strategies, Persistent Agents](https://blog.jetbrains.com/ai/2025/10/koog-0-5-0-is-out-smarter-tools-persistent-agents-and-simplified-strategy-design/)
Four notable changes: (1) Non-graph strategy API — write agent strategies directly in Kotlin code without defining graph structures, with the same state management and history compression available; good for rapid prototyping before committing to production graph architectures. (2) `RollbackToolRegistry` — enables agents to undo tool-call side effects at checkpoints, preventing inconsistent environment state after interruption. (3) `AIAgentService` — a new service layer for managing multiple agents as state-managed, single-use instances, simplifying orchestration. (4) Tool descriptor auto-generation across JVM and non-JVM targets; `subgraphWithTask` and `subgraphWithVerification` no longer require boilerplate `finishTools` or `SubgraphResult` types. The release also adds LLM-as-judge output evaluation and streaming support for tool calls — partial results can flow to a UI in real time while the tool is still running.

### [Gemini 3 Pro Arrives in Android Studio Otter](https://android-developers.googleblog.com/2025/11/gemini-3-is-now-available-for-ai.html)
Gemini 3 Pro — the model specifically engineered for coding and agentic workflows — is now built into Android Studio Otter. The free tier gets a 1M token context window, which is meaningful for Agent Mode sessions on large codebases. Add a Gemini API key in settings to unlock higher rate limits and longer sessions. Enterprise users with Gemini Code Assist licenses get access via the Google Cloud console. The model is what makes Agent Mode's multi-step autonomous work actually viable at scale.

## October 2025

### [ML Kit Prompt API: Custom On-Device Gemini Nano (Alpha)](https://android-developers.googleblog.com/2025/10/ml-kit-genai-prompt-api-alpha-release.html)
Previous ML Kit GenAI APIs were locked to specific tasks (summarize, describe image). The Prompt API opens arbitrary natural-language prompts to Gemini Nano running entirely on-device. You set temperature, topK, candidateCount, and maxOutputTokens per call — the same parameters you'd use in a server-side model. The library's current alpha caveat: it performs best on Pixel 10, which ships with Gemini Nano (nano-v3); for development without Pixel 10 hardware, Google recommends Gemma 3n (same architecture). The real-world example given: Kakao Mobility used it to extract delivery details from a plain-text message, replacing a manual copy-paste form — offline, private, no server roundtrip. Recommended use cases include image classification, document scanning with categorization, review classification, and email information extraction.

### [Microdosing AI for Mobile Dev](https://blog.mmckenna.me/microdosing-ai-for-mobile-dev)
Matt McKenna makes the case for small, targeted AI use over wholesale agent-driven development. The concrete examples: drop a screenshot into an agent to trace where a UI string comes from in the codebase (saves 10–15 minutes of manual search); use AI for a first-pass PR review before opening it to teammates (catches obvious issues, reduces back-and-forth). He's explicit that he still relies on humans for real code review — AI finds the low-hanging fruit, not the design-level problems. The throughline: every workflow described saves real daily time without requiring you to cede control of what the code actually does.

### [New Agentic Android Studio + Fall AI APIs](https://android-developers.googleblog.com/2025/10/new-agentic-experiences-for-android.html)
The October Google announcement sweep: Prompt API alpha launch, the first Android XR device (Samsung Galaxy XR), and new Play Console productivity tools. Useful as a single-page reference for everything Google shipped in the fall 2025 cycle if you missed the individual announcements.

## September 2025

### [The Kotlin AI Stack](https://blog.jetbrains.com/kotlin/2025/09/the-kotlin-ai-stack-build-ai-agents-with-koog-code-smarter-with-junie-and-more/)
JetBrains laid out a four-tier hierarchy for building AI with Kotlin: (1) raw Java/Kotlin SDKs for Anthropic and OpenAI; (2) Spring AI for model-provider abstraction with Kotlin function registration; (3) MCP Kotlin SDK for standardized tool exposure across agents; (4) Koog for full agentic workflows with persistence, checkpointing, OpenTelemetry integration (Langfuse, W&B Weave), typed streaming, and Spring Boot / Ktor connectors. Junie is the IDE-layer agent that handles full features end-to-end, architectural changes, and test execution, with a GitHub Early Access Program for issue-to-PR workflows. JetBrains also released Mellum — their in-house code-focused LLM, now open-source — with a Kotlin-specific fine-tuned variant available for community customization. The positioning is deliberate: they want Kotlin, not Python, to be the go-to language for JVM-based AI agent development.

### [Google + Kaggle AI Agents Intensive](https://blog.google/technology/developers/ai-agents-intensive/)
A free 5-day course (November 10–14, 2025) covering agent architectures, tool and memory systems, evaluation, and production deployment. Format: technical deep dives, hands-on codelabs, daily Discord discussions, and YouTube livestreams with Google engineers. Built on Google's previous GenAI Intensive, which drew 280,000 learners. Capstone projects could be featured on Google's and Kaggle's social channels. Beginner-to-advanced scope. Useful if you want a structured on-ramp to agent concepts beyond just reading framework docs.

### [Koin IDE Plugin 1.4.0: AI-Assisted DI Debugging](https://blog.kotzilla.io/koin-ide-plugin-ai-debugging-android-kotlin-development)
The plugin now generates ready-to-use AI prompts for each detected dependency injection issue — crash, performance, or configuration problem — formatted for GitHub Copilot, Gemini, Claude Code, or whichever assistant you use. For config issues, the prompt includes issue type, severity, location, parent/dependent components, and specific instructions for generating the correct Koin module declaration. The configuration tree view supports filtering by Gradle module, module type (Android/JVM/KMP), file, and declaration type — useful in large multi-module projects where manually tracing dependency definitions across files is painful. Keeps DI debugging within the IDE rather than requiring a context switch.

## July 2025

### [Llamatik: Run LLMs + Whisper + Stable Diffusion on KMP](https://github.com/ferranpons/Llamatik)
A Kotlin Multiplatform library wrapping llama.cpp (b7815), whisper.cpp (v1.8.3), and stable-diffusion.cpp behind three clean bridge APIs. `LlamaBridge` covers streaming and non-streaming text generation, JSON schema-constrained outputs (critical for structured extraction), and embedding vectors. `WhisperBridge` transcribes 16kHz mono WAV files on-device. `StableDiffusionBridge` generates images with configurable width, height, steps, and guidance scale. Supports Android (API 26+), iOS (16.6+), Desktop JVM (21+), and WASM. Published on Maven Central, no custom native toolchain setup required — just Gradle dependencies. At v0.17.0 with 148 commits, and production apps are already shipping on Play and App Store. The optional HTTP client enables hybrid local/remote architecture for when on-device inference isn't sufficient.

**The thread across all of this:** Google is systematically rebuilding Android around AI agents. AppFunctions, the Prompt API, Agent Mode's device control, and Android Bench are not independent features — they're one coherent bet that the next interaction layer is "tell Gemini what you want" rather than "open an app." For developers that means two things: expose your app's functionality to agents via AppFunctions or you risk being bypassed entirely, and adopt AI-first tooling in your dev workflow now because the gap between teams that have and haven't will compound quickly.
