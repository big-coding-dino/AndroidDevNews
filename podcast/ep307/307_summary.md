**Ep. 307 — Harness Engineering: the hard part of AI coding**
*Fragmented Podcast · Kaushik & Yuri · 30 min · 2026-03-17*

Harness engineering is the umbrella term for shaping the environment around an AI coding agent so it acts reliably — and Kaushik and Yuri argue it's now the real bottleneck, not the model itself. The framing comes from an OpenAI post about their Codex codebase: 1M+ lines, 1,500 merged PRs, zero manually written lines. The hard part wasn't generating code — it was building the scaffolding that kept agents on track. Engineers spent ~20% of their time course-correcting and encoding those corrections back into the system. The role shift: from writing code to steering agents.

The hosts structure harness shaping around five themes. **Agent legibility** means making the repo navigable by agents, not just humans — docs organized by feature and cross-cutting concerns, referenced from AGENTS.md as a map, with notes that link to other notes so agents can traverse them independently. **Closing feedback loops** means giving agents the same feedback infrastructure developers need: tests whose output is immediately actionable (not buried in a file the agent can't find), type checkers, linters, and log access. The key insight is that if a test dumps results somewhere the agent can't read, the loop isn't actually closed. This extends to screenshot tests, emulators for mobile, and paginated APIs — whatever humans use to verify the system, agents need engineered access to as well. **Persistent memory** addresses two problems: preventing repeated mistakes and picking up mid-task context. Solutions range from a `MEMORY.md` file to vector DB embeddings, with the hosts noting the space is still maturing. **Entropy control** (a mechanical engineering callback) frames the core challenge: left unattended, codebases drift toward disorder — and that drift accelerates when agents are shipping 1,000 PRs.

The episode covers only four of the five themes before cutting off — the fifth (entropy control tactics) was still mid-explanation at the transcript end.

**Why it's worth your time:** If you're using Claude Code or any agentic coding tool and wondering why results degrade over time or across sessions, this episode names the problem precisely and gives a concrete framework for addressing it. The distinction between shaping vs. building the harness is a useful mental model for deciding where to invest effort right now.
