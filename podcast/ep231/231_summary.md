**Ep. 231 — Learning Jetpack Compose with Vinay Gaba**
*Fragmented · Don Felker & Kaushik Gopal · 50 min · 2022-06-06*

Vinay Gaba (Airbnb engineer, GDE) joined to discuss Jetpack Compose from the practitioner side — not the compiler internals, but what it's like to push it hard in production and build tooling around it.

On *why Compose exists at all*: the View system's core problem is `View.java`, a 30–40K line class that every view inherits from. Composition over inheritance is well-understood best practice, but the View hierarchy made it structurally impossible. On top of that, the UI toolkit was coupled to the OS release cycle, meaning improvements to `TextView.java` shipped only with major Android releases. Compose as a standalone library ships bi-weekly. State management in the View world was also open-ended — single source of truth was the community-settled answer, but nothing in the framework enforced or facilitated it. Kotlin-first APIs were an afterthought. All of that together made a ground-up rewrite the more defensible option over just extracting the View layer into a library.

Vinay built three community resources worth bookmarking. **jetpackcompose.app** is an IFTTT-style mapping tool: select a View-system API (e.g. RecyclerView) and it returns the Compose equivalent (LazyColumn) with a linked code example. **Learn Jetpack Compose by Example** is a GitHub repo of encapsulated, heavily commented code samples — no assumed knowledge, every line explained, useful for developers who learn best by reading real code. **Showcase** is an annotation processor-based library open sourced with Airbnb — essentially Storybook for Android. Annotate your composables and it generates a browsable component catalog across your entire repo. It goes beyond Android Studio's `@Preview` (which is per-file) by letting you search components by name, see all style variants, and force-render them under RTL and dark mode without toggling any device setting.

**Why it's worth your time:** If you're still evaluating whether to adopt Compose or just starting out, this episode delivers the architectural rationale for *why* the rewrite happened and a practical map of community resources that compress the learning curve significantly. Vinay's tooling — especially Showcase — is relevant even for teams already using Compose who are managing component sprawl across large engineering orgs.
