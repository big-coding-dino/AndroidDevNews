**Ep. 305 — Subagents explained: What they are, when (not) to spawn them**
*Fragmented · Kaushik & Yuri · 27 min · Feb 16, 2026*

Kaushik and Yuri unpack the overloaded term "agent" before getting to the actual subject: subagents as a precise technical construct. They distinguish three common usages — agents.md (just a system instructions file), agent SDKs (Langchain, OpenAI Agent SDK, Anthropic Agent SDK for building your own agentic systems), and full products like Open Claw — before settling on the definition that matters here.

A subagent is a fresh context window with a specific system prompt and scoped tool access. That's the canonical definition, and each piece matters. "Fresh context window" means it launches without your existing conversation history — isolated by design. The system prompt lets you define a completely different persona and focus for the execution unit. Scoped tool access means you can limit it to only the tools it needs (e.g., a research agent gets Perplexity MCP and browser tools, nothing else), preventing context bloat and constraining what it can do. Under the hood, a subagent is literally a tool call — the `task` tool — and the main agent decides when to invoke one based on the subagent's description metadata.

The hosts also clarify the modes vs. subagents distinction: modes (plan, build) change the system instructions of the *main* agent, whereas subagents are spawned as separate execution units that run in parallel. The RTS game analogy lands well — your main agent is the base, subagents are the scouts sent into the fog of war. You don't bring your entire army to explore the map; you send cheap peons in multiple directions, they return with a summary, and your main agent reasons on the intel. This is exactly why the November 2025 Claude Code / Opus 4.5 release was a step change: plan mode started spawning multiple Explorer subagents to gather codebase context in parallel, returning summaries rather than raw logs.

Key practical notes for Claude Code: custom subagents live in an `agents/` directory as markdown files with YAML front matter. You can pin a cheaper model (Haiku instead of Opus) per subagent. Three built-in subagents ship with Claude Code — Explorer, Plan, and General Purpose — each with different default tool sets. If no custom or built-in agent matches, the subagent inherits the model from the main session.

**Why it's worth your time:** If you've been treating subagents as a vague "AI does parallel stuff" feature, this episode gives you the mental model to use them intentionally — right model, right tools, right scope — so your main agent stays lean and your spawned units actually stay focused.
