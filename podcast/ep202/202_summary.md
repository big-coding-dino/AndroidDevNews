**Ep. 202 — Dagger on the Anvil with Ralf Wondratschek**
*Fragmented · Don Felker & Kaushik Gopal · 77 min · 2021-02-01*

Ralf Wondratschek, Square's technical lead on the Developer Empowerment team and creator of Anvil, joins the show to explain how Anvil reduces Dagger 2 boilerplate and how Square structures its massive Android codebase to enable fast isolated feature development.

Dagger 2's core constructs are **modules** (which provide and construct dependency instances) and **components** (the graph interface through which you call `@Inject` to receive them). The pain point Anvil solves: when you're wiring up a development app — a stripped-down sandbox app for iterating on a single feature — you have to add both the Gradle dependency on a feature's wiring module *and* manually add that feature's Dagger module to your component in Kotlin code. With dozens of features, this double-declaration becomes tedious and error-prone. Anvil automatically discovers and registers Dagger modules from imported wiring modules, so adding the Gradle dependency is sufficient — the module wiring happens automatically via an annotation processor.

The broader context is Square's **isolated development** approach. Their main app grew so large that even three-minute incremental builds killed productivity. Square enforces a strict three-module structure for every library: a public API module (interfaces only), an implementation module (concrete code), and a wiring module (Dagger bindings). Dependencies are allowed only upward toward public APIs — CI checks enforce this mechanically. This clean layering lets development apps swap real implementations for no-op or fake implementations, dramatically reducing compile times. The tradeoff is IDE overhead: loading hundreds of modules strains Android Studio and IntelliJ, and Square is working with Google and JetBrains to address that bottleneck.

Anvil started as a practical solution to the Gradle sync and module-wiring problem, not as a grand library project. Ralf built it while at Square, initially as an IC, then continued after transitioning into a tech lead manager role. The library generator IntelliJ plugin that creates the three-module boilerplate in two clicks was a prerequisite — without it, the isolated development workflow would have been too burdensome for developers to adopt consistently.

Why it's worth your time: If you're managing a multi-module Android app with Dagger and sick of manual module registration, Anvil eliminates that boilerplate. More importantly, Square's isolated development pattern — even if you don't adopt it fully — is a useful mental model for thinking about how to keep large-codebase development fast by enforcing clean dependency boundaries between features.
