**Ep. 245 — Treehouse, Redwood and Zipline with Colin White**
*Fragmented Podcast · Don Felker & Kaushik Gopal · 55 min · 2023-05-15*

Colin White (creator of Coil, now at Cash App/Block) returns to explain Treehouse — an internal Cash App codename for a cross-platform UI system built on top of two open source libraries: Redwood and Zipline.

The episode starts by drawing a clear line between Jetpack Compose (the compiler + runtime that handles state diffing) and Jetpack Compose UI (the widget layer that renders to a canvas). That distinction is the conceptual foundation for everything else. Redwood slots itself between those two layers: it intercepts the diffs the Compose compiler produces and routes them to any UI toolkit — Android Views, UIKit on iOS, HTML elements on the web, or Compose UI itself. The mechanism is schema-driven. You define your widget types (e.g. `Button`, `Text`) and their properties in Kotlin, feed that to the Redwood compiler plugin, and it generates two things: composable functions that look like standard Compose UI calls, and interfaces with per-property callbacks that your target UI toolkit implements. When a property changes, the interface method fires and you update whatever native widget you're backing it with. Redwood doesn't ship a built-in schema — you're expected to model your own design system — though it does provide layout primitives (rows, columns) so you don't have to reinvent those. Zipline (covered later in the episode) handles the dynamic code delivery side of Treehouse, enabling server-driven UI updates without app store releases.

**Why it's worth your time:** If you've used Compose but never thought carefully about where the compiler ends and the UI toolkit begins, this episode makes that architecture concrete and shows why it matters. The Redwood schema model is a practical template for anyone building a shared design system across Android, iOS, and web using Kotlin Multiplatform.
