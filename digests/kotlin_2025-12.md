### [ComposeGuard](https://github.com/AndroidPoet/compose-guard)

Real-time Compose best-practices enforcement as an IntelliJ/Android Studio plugin. Covers naming (PascalCase/camelCase), modifier rules (required parameter, ordering, no `Modifier.composed`), state (`remember`, hoisting, `mutableStateOf` types), parameter ordering, trailing lambda convention, and more. Instant feedback with gutter icons and quick fixes — no build step.

### [A Better Way to Explore kotlinx-benchmark Results with Kotlin Notebooks](https://blog.jetbrains.com/kotlin/2025/12/a-better-way-to-explore-kotlinx-benchmark-results-with-kotlin-notebooks/)

Kotlin Notebooks can load `kotlinx-benchmark` text output, parse it with the DataFrame API, and render interactive charts — turning raw benchmark scores into explorable visualisations. The workflow: run your benchmark suite → load output into a notebook → use `DataFrame.read()` to parse → plot with the Kandy or lets-plot integration. Useful for comparing benchmark results across hardware configurations or before/after an optimisation.

### [How Backend Development Teams Use Kotlin in 2025](https://blog.jetbrains.com/kotlin/2025/12/how-backend-development-teams-use-kotlin-in-2025/)

Top anti-patterns in self-taught backend Kotlin teams: deep inheritance hierarchies (use sealed classes + data-oriented design instead), overuse of `?` for nullable types (use non-null at internal boundaries), and swallowing `CancellationException` in catch blocks (always rethrow). The article pairs each anti-pattern with a concrete idiomatic fix and explains the underlying type system or runtime reasoning.

---

### [How Mobile Development Teams Use Kotlin in 2025](https://blog.jetbrains.com/kotlin/2025/12/how-mobile-development-teams-use-kotlin-in-2025/)

The mobile-focused companion post homes in on the single most common production coroutine bug: swallowing `CancellationException` inside `catch(e: Exception)` or `runCatching`, which silently breaks cooperative cancellation. Fix: always rethrow `CancellationException`, use `coroutineContext.ensureActive()` in loops, and consider the `detekt` `SwallowedException` rule. Also covers `supervisorScope` for parallel task isolation and structured scope design for `ViewModel` coroutines.

### [Minimal Setup for Compose Desktop](https://kt.academy/article/compose-desktop)

A stripped-back Compose Desktop project setup — just `kotlin("jvm")`, the `compose` plugin, `compose.hot-reload`, and the `application` plugin. No multiplatform module structure, no complex source sets. The article targets developers who want to add a desktop UI to scripts or build a minimal tool without the full CMP project template. Uses Compose Desktop 1.10.0-beta01 and Kotlin 2.2.20.

### [Jindong: Declarative Haptic Feedback for Compose Multiplatform](https://github.com/compose-jindong/jindong)

A Compose Multiplatform library for defining haptic patterns with a declarative DSL mirroring how you write UI: `Jindong(trigger) { Haptic(100.ms); Delay(50.ms); Haptic(50.ms, HapticIntensity.STRONG) }`. Supports Android and iOS via a unified API. Add via `io.github.compose-jindong:jindong-compose`. "Jindong" (진동) is Korean for "vibration."

### [Setting Up Kotest on Kotlin Multiplatform](https://alyssoncirilo.com/blog/kotest-kmp-setup/)

Step-by-step Gradle configuration for Kotest across iOS, Linux, JVM, and Android targets on a KMP project. Requires the `io.kotest` Gradle plugin and `kotest-framework-engine` in `commonTest`. iOS and Linux work immediately; JVM and Android need the `kotest-runner-junit5` dependency in their respective source sets. Full template on GitHub at `Kotest-KMP-Setup`.
