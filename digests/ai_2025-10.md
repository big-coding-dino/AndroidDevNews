### [How to Use the Develocity IntelliJ Plugin to Speed Up Gradle Builds](https://gradle.com/blog/develocity-intellij-plugin-speed-up-gradle-builds-insights/)

The Develocity IntelliJ Plugin brings real-time Gradle build analysis directly into IntelliJ IDEA and Android Studio, targeting the "inner development loop" — the cycle from making a change to getting back to a green build. The standout feature is a Live Build Timeline that visualizes task execution and highlights bottlenecks without leaving the IDE. The plugin integrates with Gradle's Build Scan infrastructure, so it can surface the same insights normally available on the Develocity server but inline during local development. Requires a Develocity server (self-hosted or Gradle's hosted offering) to unlock the full feature set.

### [Sweep Is the Best Autocomplete for Android Studio](https://sweep.dev/?utm_source=androidweekly)

Sweep brings Cursor-style next-edit autocomplete to JetBrains IDEs including Android Studio, using its own LLMs rather than routing through third-party providers. The Tab model predicts context-aware multi-line code changes in milliseconds, taking into account recent edits and Kotlin type information. It's currently rated 4.9 stars with 40k+ installs on the JetBrains Marketplace and supports all JetBrains IDEs. *(Summary-only — content was mostly marketing copy.)*

### [The Ktor Roadmap and Previews](https://blog.jetbrains.com/kotlin/2025/09/ktor-roadmap-2025/)

Ktor 3.3.0 ships OpenAPI documentation generation and WebRTC support in early preview, with OpenAPI using compile-time analysis (via the KLIP process) to automatically produce an API model from your route definitions — no runtime reflection required. Service discovery is also on the near-term roadmap. The KLIP (Ktor Library Improvement Process) is fully public on GitHub, giving the community visibility into upcoming proposals before they land. Use `start.ktor.io` to scaffold new projects with the latest plugins already configured.

### [Build a Kotlin Multiplatform Project and Win a Trip to KotlinConf 2026](https://blog.jetbrains.com/kotlin/2025/10/kotlin-multiplatform-contest-2026/)

JetBrains has opened the 2026 KMP Contest for students and recent graduates, with the top three projects winning full trips to KotlinConf 2026 in Munich (May 21–22). Submissions can target Android, iOS, desktop, web, or server and are encouraged — though not required — to incorporate AI features from the Kotlin AI Examples repository. The submission deadline is January 12, 2026, with winners announced shortly after. All valid entries receive Kotlin merchandise.

### [Boost User Engagement with AI Image Generation](https://android-developers.googleblog.com/2025/10/boost-user-engagement-with-ai-image.html)

Firebase AI Logic now exposes two complementary image generation models: Imagen (in developer preview) with inpainting and outpainting via mask-based editing, and Gemini 2.5 Flash Image ("Nano Banana") in general availability for conversational, context-aware image generation. The two serve different use cases — Imagen is better suited for precise pixel-level editing tasks, while Gemini 2.5 Flash Image is ideal for dynamic illustrations that need to align with the user's current in-app context. Both support iterative natural-language editing so users can refine outputs conversationally. Before shipping, review the AI safety guidance from Firebase, especially for user-generated content flows.

### [Build Fire TV Apps with React Native for Vega](https://fandf.co/4aIHRkR)

Amazon's Vega SDK lets developers write a single codebase that deploys across all supported Fire TV and Vega OS-powered devices, and React Native for Vega (RNV) adds React Native support on top of it. RNV supports remote control, touch, and voice interaction patterns, with built-in focus management for TV navigation — a common pain point in lean-back experiences. The implementation targets RN API parity, so existing React Native knowledge transfers. *(Summary-only — limited technical depth in available content.)*

### [Jetliner XR: Spatial Development on Android](https://charlesmuchene.com/jetliner-xr)

A hands-on exploration of Android XR using a 3D A320neo jetliner demo, covering the three core platform capabilities: UI spatialization (positioning composables in 3D space), SceneCore for loading and manipulating 3D models, and ARCore for understanding the physical environment. The demo transitions between Home Space (2D coexistence mode) and Full Space (immersive exclusive mode), illustrating when to use each. Samsung's Galaxy XR (Project Moohan) shipped in October 2025 as the first Android XR device, making this a practical starting point for developers evaluating the platform.

### [Bringing Androidify to XR with the Jetpack XR SDK](https://android-developers.googleblog.com/2025/10/bringing-androidify-to-xr-with-jetpack.html)

Google's open-source Androidify app was ported to Android XR using the Jetpack XR SDK, demonstrating how adaptive Compose layouts and reusable composables translate into spatial experiences without a full rewrite. The XR version uses Gemini for avatar generation via CameraX input, Navigation 3 for spatial navigation, and Jetpack XR's panel and subspace APIs to position UI in 3D. The key insight: apps already built with proper adaptive layouts (for phones, foldables, tablets) have a significantly lower barrier to entering XR. The source is open on GitHub and intended as a reference for the transition.

### [Getting Started with Unity and Android XR](https://android-developers.googleblog.com/2025/10/getting-started-with-unity-and-android.html)

Android XR is built on OpenXR and Vulkan open standards, and Unity 6 gains first-class support through official packages: the XR Interaction Toolkit, OpenXR plugin, and Android XR-specific extensions. The setup guide covers environment configuration, the package ecosystem, input methods (gaze, hand tracking, controller), privacy/permissions requirements, and composition layers for rendering efficiency. Unity's cross-platform XR support means a single project can target Android XR and other OpenXR-compliant devices. The Vulkan-only rendering path is required — OpenGL ES is not supported on Android XR.

### [Microdosing AI for Mobile Dev](https://blog.mmckenna.me/microdosing-ai-for-mobile-dev)

A practical catalogue of small, high-ROI AI integrations into daily Android development — not vibe-coding, but targeted task offloading. Highlighted patterns: dropping a screenshot into an AI agent to reverse-lookup where UI text is rendered (replaces manual string/ViewModel/Composable tracing); using an agent for a first-pass PR review before opening to teammates (catches missed overrides and readability issues); asking an AI to generate boilerplate Kotlin data classes from JSON samples. The key framing is "accelerate the in-between steps" — the tedious connective tissue of development, not the core problem-solving. Each use case takes seconds to initiate and provides immediate, concrete output.

### [Introducing Cahier: A New Android GitHub Sample for Large Screen Productivity](https://android-developers.googleblog.com/2025/10/introducing-cahier-new-android-github.html)

Cahier ("notebook" in French) is a new open-source Android sample app optimized for tablets and foldables, built specifically to showcase the Ink API now entering beta. The app demonstrates combining text, freehand drawing, and images in a single note — a realistic test case for the Ink API's pen/stylus input, pressure sensitivity, and stroke rendering. Google apps like Docs, Photos, Chrome PDF, and Circle to Search all use the same underlying Ink APIs. Key adaptive UI patterns in the sample include multi-pane layouts, drag-and-drop between panes, and keyboard/mouse support for desktop-class interactions.

### [Swift Android Gradle Plugin: Build Swift Libraries for Android](https://charlesmuchene.com/swift-android-gradle-plugin)

Following the Swift for Android working group's announcement that Swift now officially supports Android, this Gradle plugin automates the cross-compilation pipeline — invoking the Swift toolchain, the new Swift SDK for Android, and the NDK in the correct sequence via standard Gradle task configuration. The demo app uses the plugin to generate fractal images in Swift and render them with Jetpack Compose, showing the interop boundary cleanly. Setup requires the Swift toolchain, the Swift Android SDK, and Android NDK; the plugin handles the rest. This is still early-stage — expect rough edges in toolchain compatibility across OS versions.

### [New Agentic Experiences for Android Studio, New AI APIs, and More!](https://android-developers.googleblog.com/2025/10/new-agentic-experiences-for-android.html)

Google's Fall Android Show announced a wave of AI updates: the Gemini Nano **Prompt API** is now in Alpha, giving developers free-form prompt access to the on-device model rather than being locked to pre-built task APIs (summarization, proofreading, image description). Android Studio gains agentic Gemini features for autonomous coding tasks. The first Android XR device (Samsung Galaxy XR) launched alongside the show. Kakao's production use case — extracting delivery details from a single natural-language message to replace a manual copy-paste form — illustrates the practical pattern for the Prompt API: replacing structured form input with unstructured natural language processed on-device.

### [ML Kit's Prompt API: Unlock Custom On-Device Gemini Nano Experiences](https://android-developers.googleblog.com/2025/10/ml-kit-genai-prompt-api-alpha-release.html)

The ML Kit GenAI Prompt API (Alpha) opens up Gemini Nano on Android to arbitrary natural language and multimodal (text + image) requests, replacing the previous task-specific APIs that only covered summarization, proofreading, and image description. Processing runs entirely on-device via Gemini Nano, so user data never leaves the device and the feature works offline. Key use cases: image classification for social media drafts, intelligent document scanning (combining a traditional ML model for OCR with Gemini for semantic understanding), and complex data transformations. Currently in Alpha — API surface may change, and availability is limited to flagship devices with Gemini Nano support.

### [New Tools and Programs to Accelerate Your Success on Google Play](https://android-developers.googleblog.com/2025/10/new-tools-and-programs-to-accelerate.html)

Google Play Console gains two AI-powered tools: a built-in emulator for deep link validation (letting you test a deep link's end-user experience directly in Play Console without a physical device), and Gemini-powered automated localization that translates new app bundles into selected languages at no cost. The translation service auto-runs on every new bundle upload, with a preview emulator and manual override available if you want to adjust or reject specific strings. Both tools target common friction points in the release pipeline — deep link bugs and localization lag — that typically require external tooling or manual QA.
