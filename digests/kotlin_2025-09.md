# Kotlin — September 2025 Digest
*Written by Claude · Source: Android Weekly*

---

### [EasyAndroidPermissions](https://github.com/ivamsi/easyandroidpermissions)

A lightweight library (`io.github.ivamsi:easyandroidpermissions:1.0.2`) that bridges `ActivityResultContracts` and Kotlin coroutines, replacing callback-based permission handling with `suspend` functions. Works in Activities, Fragments, and Compose; handles concurrent requests correctly and cleans up resources on lifecycle end. The coroutine-first API means permission checks fit naturally into `viewModelScope` or `lifecycleScope` flows without nesting callbacks.

### [Pragmatic KMP for Mobile – Part 4: Navigation](https://kiranrao.in/blog/pragmatic-kmp-part-04/)

Somnox's engineering team shows how to incrementally introduce KMP navigation alongside existing platform navigation: Android used `NavController` (Nav2 era), iOS used `NavigationView`. Part 4 covers two cases — navigating between multiplatform screens and from a multiplatform screen to a native screen. The solution uses platform-specific expect/actual declarations for navigation callbacks, letting the KMP code trigger navigation events that the platform layer handles natively. A pragmatic middle-ground that avoids forcing a full navigation migration upfront.

### [Kotlin 1.6.20 Released](https://blog.jetbrains.com/kotlin/2022/04/kotlin-1-6-20-released/)

Kotlin 1.6.20 makes **hierarchical project structure the default** for multiplatform projects, eliminating the opt-in flag. Context receivers land as a preview for Kotlin/JVM. The JVM IR backend gains parallel compilation within a single module — a meaningful build speed improvement for large projects. Incremental compilation arrives in the Kotlin/JS IR backend. Kotlin/Native gets performance improvements. *(Note: article dated 2022 but surfaced via Android Weekly in Sept 2025.)*

### [Accessing Native macOS APIs in Compose Multiplatform via JNI](https://www.marcogomiero.com/posts/2025/compose-desktop-macos-api-jni/)

Compose Multiplatform desktop apps run on the JVM, which means macOS-native APIs (like iCloud/CloudKit) aren't accessible by default. Marco Gomiero's solution from his FeedFlow RSS reader: write the macOS integration in Objective-C/Swift, compile it as a native library, and call it from Kotlin via JNI. The setup involves a Gradle task that compiles the native code and copies the `.dylib` into the app bundle at build time. The approach is unavoidable for any CMP desktop app that needs OS-level integration, and the article documents the full Gradle + JNI wiring needed.

### [Building a Cross-Platform Invoice App with Compose Multiplatform](https://www.paleblueapps.com/rockandnull/billin-building-a-cross-platform-invoice-app-with-kotlin-multiplatform/)

Billin chose CMP over a separate SwiftUI app to share the full UI layer across Android and iOS, not just business logic. Key technical decisions: shared PDF preview via a Compose canvas renderer, shared payment and client management state, and a single navigation graph. The post is honest about where CMP required extra work — platform-specific file sharing and some iOS-only UI adjustments — but frames these as manageable. The conclusion: for a team already invested in Kotlin and Compose, CMP is a faster path to iOS than SwiftUI.

### [The Kotlin AI Stack: Koog, Junie, MCP](https://blog.jetbrains.com/kotlin/2025/09/the-kotlin-ai-stack-build-ai-agents-with-koog-code-smarter-with-junie-and-more/)

JetBrains maps out four levels of Kotlin AI integration: raw SDK calls (OpenAI/Anthropic SDKs, both Kotlin-under-the-hood), high-level libraries (Spring AI, LangChain4j), MCP tool exposure via the official MCP Kotlin SDK, and full agent frameworks (Koog). Koog is KMP-compatible — agents run on JVM, Android, JS/WasmJS, and iOS. Mellum, JetBrains' Kotlin-fine-tuned LLM powering Junie, is now open-source for community tooling. The post also covers why Kotlin is well-suited for AI workloads: Java interop for existing SDKs, multiplatform for sharing agent logic, and concise syntax for graph-based agent strategies.
