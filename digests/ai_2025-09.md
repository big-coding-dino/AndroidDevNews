### [Android 13 Deep Dive: Every Change Up to DP2, Thoroughly Documented](https://blog.esper.io/android-13-deep-dive/)

Mishaal Rahman's exhaustive changelog of Android 13 (API 33) covers every behavioural change, new API, and UI update through DP2. Key additions include `isAudioDescriptionRequested()` in `AccessibilityManager` for detecting whether users prefer audio description tracks, new Quick Settings tiles for color correction and one-handed mode, and full Bluetooth LE Audio support with an LC3 encoder/decoder in the Bluetooth stack. Bluetooth LE Audio introduces Broadcast Audio (one-to-many streaming), lower power consumption, and better audio quality — with no app changes required to benefit. Platform Stability was reached at Beta 3 (June 2022), so all APIs are now finalized; Beta 4 allowed app publishing targeting API 33.

### [Running a RAG Powered Language Model on Android Using MediaPipe](https://darrylbayliss.net/running-a-rag-powered-language-model-on-android-using-mediapipe/)

A step-by-step guide to implementing an on-device RAG pipeline using MediaPipe, Gemma3-1B (`gemma3-1B-it-int4.task`), and the Gecko embedder (`Gecko_256_f32.tflite`). The setup requires pushing the model and embedder files to the device via `adb`, then using `localagents-rag:0.3.0` alongside `tasks-genai:0.10.27` to create an SQLite vector store for retrieval. Chunking strategy (using `<chunk_splitter>` tags in source text) directly affects RAG quality — experimentation is strongly encouraged. A key caveat: the language model, embedder, and tokenizer together consume several gigabytes on-device, so plan your storage strategy carefully before shipping.

### [Google and Kaggle Launch AI Agents Intensive Course](https://blog.google/technology/developers/ai-agents-intensive/)

Google and Kaggle are running a free 5-Day AI Agents Intensive course (November 10–14), designed by Google's own AI researchers and engineers. The curriculum covers agent architectures, tools, memory, and evaluation — progressing from simple single agents to multi-agent systems — with hands-on codelabs and live Discord/YouTube sessions each day. A capstone project gives participants a shot at prizes and featured exposure on Google/Kaggle channels. This follows the previous GenAI Intensive that drew 280,000 learners; registration is required in advance.

### [Koin IDE Plugin 1.4.0: AI-Powered Dependency Injection Debugging](https://blog.kotzilla.io/koin-ide-plugin-ai-debugging-android-kotlin-development)

Koin IDE Plugin 1.4.0 adds AI-assisted debugging directly inside Android Studio and IntelliJ IDEA, generating context-aware prompts for each detected DI issue that can be pasted into Copilot, Gemini, Claude Code, Junie, or any other AI assistant without switching tools. The Application Issues and Configuration Issues tabs now emit prompts that include issue type, severity, affected components, likely root causes, and suggested fix syntax including imports and Koin module declarations. Additional improvements include Gradle-module filtering, platform filtering (Android/JVM/KMP), two-way editor sync, and better performance on large codebases. Compatible with IntelliJ IDEA 2024.2+, Android Studio Ladybug+, Kotlin 1.7.0+, and Android API 21+.

### [The Kotlin AI Stack](https://blog.jetbrains.com/kotlin/2025/09/the-kotlin-ai-stack-build-ai-agents-with-koog-code-smarter-with-junie-and-more/)

JetBrains outlines the full Kotlin AI stack across four levels: raw Java SDKs (OpenAI/Anthropic, both written in Kotlin), high-level libraries like Spring AI and LangChain4j, MCP tool exposure via the official MCP Kotlin SDK, and full agentic frameworks — primarily Koog. Koog is now open-source, multiplatform (JVM, Android, JS/WasmJS, iOS), and provides persistence/checkpointing, OpenTelemetry observability (Langfuse, W&B Weave), structured output, typed streaming, smart retries, Spring Boot and Ktor plugins, and multi-model routing. Mellum, JetBrains' in-house LLM powering Junie, is also now open-source and fine-tuned specifically for Kotlin, opening the door for community-built tooling on top of it. Other emerging Kotlin agent frameworks include Arc (Deutsche Telekom), Embabel (Spring + LangChain4j), and Spring AI Alibaba.

### [Let's Talk Security: Answering Your Top Questions About Android Developer Verification](https://android-developers.googleblog.com/2025/09/lets-talk-security-answering-your-top.html)

Google is introducing mandatory developer verification for distributing apps to certified Android devices, with the requirement taking effect more than a year from announcement to allow adaptation time. Crucially, the change does not affect `adb`-based deployments — Android Studio builds, local debugging, and emulator testing are completely unaffected. Developers who distribute APKs to testers through any channel other than `adb` (including Google Play Internal Testing, Firebase App Distribution, or third-party tools) will need to verify identity and register the package. Early access invitations begin in October 2025; signing up in advance is recommended to avoid friction at launch.
