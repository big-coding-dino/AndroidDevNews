### [Update Your Kotlin Projects for Android Gradle Plugin 9.0](https://blog.jetbrains.com/kotlin/2026/03/agp-9-0-kotlin-migration/)

AGP 9.0 introduces two breaking changes for Kotlin projects. First, the Kotlin Android plugin (`kotlin("android")`) is replaced by AGP's built-in Kotlin support — remove the separate Kotlin Android plugin from your `build.gradle.kts` and let AGP handle Kotlin compilation. Second, KMP projects targeting Android must migrate to the new Android KMP library plugin (`com.android.kotlin.multiplatform.library`). Both changes require Gradle and AGP 9.0+ and affect all existing Android Kotlin projects; the post links to the official migration guides.

### [Room 3.0: First Alpha — Modernizing Room for KMP](https://android-developers.googleblog.com/2026/03/room-3-0-alpha.html)

Room 3.0 alpha is a major breaking version focused on Kotlin Multiplatform, adding JavaScript and WebAssembly targets on top of existing Android, iOS, and JVM desktop support. Breaking changes include: the `@Database` annotation now requires explicit KMP configuration, platform-specific drivers replace the monolithic Room runtime, and the query API is refactored around Kotlin suspend functions as first-class citizens. The post outlines a migration path from Room 2.x and explains the architectural reasoning behind each breaking change. For apps already using Room on Android, plan the migration carefully — it's a significant API surface change.

### [Unidirectional Data Flow as a Functional Declaration](https://medium.com/@tjdahunsi/udf-functional-declaration)

A conceptual reframing of UDF architecture: state production pipelines are functions from a stream of inputs (user events, external data) to a stream of output states. This functional view makes the relationship between events and state explicit in the type system — rather than imperative mutation scattered across a ViewModel, you declare a pure transformation. The post explores how this model maps to `Flow` operators and `StateFlow`, and why functional UDF makes testing straightforward (give it an input stream, assert on the output stream). *(Note: article originally from 2022, resurfaced in Android Weekly March 2026.)*
