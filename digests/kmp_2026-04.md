# Kotlin Multiplatform — April 2026 Digest
*Written by Claude · Source: Android Weekly*

### [An update on Android KTX](https://jakewharton.com/an-update-on-android-ktx/)
After eight years, Jake Wharton announced the wind-down of the **Android KTX** (and all `-ktx`) extension libraries — not because they failed, but because they succeeded. Kotlin adoption across the AndroidX ecosystem has been thorough enough that all KTX extensions have been merged directly into their respective main libraries, making the `-ktx` modules empty and obsolete. The KTX story started with `core-ktx` co-authored by Wharton, Chris Banes, and Romain Guy, and expanded into dozens of libraries — `fragment-ktx`, `lifecycle-viewmodel-ktx`, `work-runtime-ktx`, and more — as a way to bolt Kotlin-friendly APIs onto Java-first AndroidX libraries. The full deprecation table covers 27 `-ktx` modules, with `activity-ktx` obsolete in 1.9.0, `core-ktx` in 1.19.0-alpha01, and `fragment-ktx` in 1.9.0-alpha01. If your project still declares `-ktx` artifacts explicitly, those dependencies are now dead weight — the Kotlin APIs they provided live directly in the base artifacts, so you can drop the `-ktx` suffix from those Gradle dependencies without losing any functionality.

### [cw-json](https://commonsware.com/blog/2026/04/07/announcing-cw-json.html)
Mark Allison (CommonsWare) has published **cw-json**, his first Kotlin Multiplatform library, marking a return to open-source library publishing after a long hiatus. The library is a collection of JSON-related artifacts, with the initial release focused on **JSON Pointer** support. Two artifacts are currently available: **`com.commonsware.json:kmp-jsonpointer`** — a pure KMP implementation of the JSON Pointer spec, providing a `JsonPointer` type with functions for creating and manipulating pointers — and **`com.commonsware.json:kmp-jsonpointer-kxs`** — an extension that integrates JSON Pointer with **Kotlin Serialization**, letting you read and mutate `JsonObject` and `JsonArray` values using pointer-based access. JSON Pointer (RFC 6901) is a standard way to reference specific values inside a JSON document using path expressions like `/users/0/name`, and having a KMP-native implementation that integrates directly with `kotlinx.serialization` types fills a real gap for teams building shared business logic across Android, iOS, and other platforms.

### [Kotlin Multiplatform @ Google](https://developer.android.com/kotlin/multiplatform)
Google officially supports **Kotlin Multiplatform (KMP)** for sharing business logic between Android and iOS, and declares it stable and production-ready. A significant portion of the **Jetpack library ecosystem** has been migrated to KMP, including `annotation` and `collection` (April 2026), `compose` (April 2026), `datastore`, `lifecycle`, `viewModel`, `viewModel-compose`, `navigation`, `savedstate` (March 2026), and `navigation3`, `navigationevent`, `paging`, `sqlite` (April 2026), with `room` (November 2025). The **Kotlin Multiplatform Plugin** for the IDE ships with a new project wizard, preflight environment checks, unified run/debug/test configurations for Android and iOS, and basic Swift support including cross-language debugging and navigation. Platform tiers define support levels: **Tier 1** covers Android, JVM, and iOS; **Tier 2** adds macOS and Linux; **Tier 3** includes watchOS, tvOS, Windows, JavaScript, and WASM. The Jetpack library migration removes the biggest practical blocker to adopting KMP — with these libraries now KMP-ready, teams can share substantial slices of production architecture (persistence, state management, navigation) across Android and iOS without rewriting in a foreign language or giving up familiar APIs.
